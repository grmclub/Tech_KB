##list commonly used productivity frameworks with a short summary

Here are the most popular productivity frameworks used globally to manage time, optimize focus, and execute tasks efficiently:

1. Getting Things Done (GTD)
Created by David Allen, this framework offloads mental clutter into an external system so you can focus entirely on execution. [1, 2, 3, 4]
    • Capture: Collect everything that catches your attention in an inbox.
    • Clarify: Process what each item means and decide if it is actionable.
    • Organize: Put tasks into specific lists (e.g., Next Actions, Projects, Waiting For).
    • Reflect: Review your lists regularly to update tasks and stay on track.
    • Engage: Choose which task to work on based on your current context and energy. [1, 2, 3, 4, 5]

2. Time Blocking / Time Boxing
A calendar-centric strategy popularized by Elon Musk and Cal Newport that treats time as a fixed, finite resource. [1, 2, 3, 4]
    • Estimate: Determine how long a specific task or project will take.
    • Block: Dedicate a specific, visual block of time on your calendar for that task.
    • Protect: Work only on that designated task during the scheduled block.
    • Box (Alternative): Set a strict time limit (box) where the work must stop, regardless of completion. 

3. The Pomodoro Technique
Developed by Francesco Cirillo, this framework uses timed intervals to maintain high mental focus and prevent burnout. [1, 2, 3]
    • Choose: Pick a single task you want to focus on.
    • Work: Set a timer for 25 minutes and work with zero distractions.
    • Break: Stop immediately when the timer rings and take a 5-minute break.
    • Repeat: After completing four work intervals, take a longer 15-to-30-minute break. [1, 2, 3]

4. The Eisenhower Matrix
A prioritization framework popularized by US President Dwight D. Eisenhower to separate urgent tasks from important ones. [1, 2, 3]
    • Do First: Handle tasks that are both urgent and important immediately.
    • Schedule: Set a specific time to do tasks that are important but not urgent.
    • Delegate: Pass tasks that are urgent but not important to someone else.
    • Eliminate: Delete tasks that are neither urgent nor important from your list. [1, 2, 3, 4, 5]

5. Agile / Scrum (Personal Kanban) [1, 2, 3]
Adapted from software development, this framework uses visual boards to manage personal task capacity and workflow. [1, 2, 3, 4, 5]
    • Backlog: List every task you need to complete eventually.
    • To Do: Move a select few high-priority tasks into this column for the current week.
    • Doing (WIP): Move tasks here when actively working on them, limiting this column to 1–2 items.
    • Done: Move completed items here to visually track your velocity and progress. [1, 2, 3, 4, 5]
